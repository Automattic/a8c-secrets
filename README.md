This repository contains the implementation of the Rust tool that we are using to encrypt/decrypt secret files from mobile and desktop repos at Automattic.

> [!NOTE]
> The secrets this tool encrypts are only meant to be secret files needed by Automatticians to compile the apps locally, as well as for our CI. For example, this include files like `secret.properties` (Android repos) / `Secrets.swift` (iOS repos) that contains API secrets for the APIs those mobile apps communicate with, or debug signing keys to allow developers to test features with signed debug builds (which is a different signing identity that the ones we use for the App Store / Play Store builds).
>
> For secrets that are only needed by the CI but not by developers (e.g. store signing identity to codesign a release build for App Store / Play Store from CI, or API Key needed to upload builds to App Store Connect / Play Store Console) will NOT be encrypted in the repo and thus not go through this `a8c-secrets` tool, but instead be provided as secrets only to CI (e.g. Buildkite secret ENV vars)

 - The encrypted files generated by this tool will live in a `.a8c-secrets/` directory at the top-level of the repos that will need those secrets, with the extension `.age`
 - The tool will always decrypt the files outside the working copy of the repository, namely in `$HOME/.a8c-secrets/<repo-slug>/`, to ensure that those decrypted secrets don't accidentally get commited to the remote (even with a `.gitignore` a typo can happen, so we don't want to take any risk) and also won't be accessible by AI Agents that you might run from / would be restricted to the working copy of the repo.
 - The underlying encryption format is using `age` - See https://github.com/C2SP/C2SP/blob/main/age.md
